║  NODE 4 — relevance_filter_node                                              ║
║  Tools   : Chroma(query_similar)                                             ║
╠══════════════════════════════════════════════════════════════════════════════╣
║                                                                              ║
║  For each raw article:                                                       ║
║                                                                              ║
║  STEP 1 — Profile similarity                                                 ║
║    Chroma → query_similar(                                                   ║
║      collection = profile_vectors,                                           ║
║      query_text = title + " " + summary,                                     ║
║      n = 3                                                                   ║
║    )                                                                         ║
║    → base_score    = top similarity score (0.0–1.0)                          ║
║    → top_chunks    = top 3 matching profile chunks                           ║
║                      (carried forward to reasoning node)                     ║
║                                                                              ║
║  STEP 2 — Feedback penalty                                                   ║
║    Chroma → query_similar(                                                   ║
║      collection = feedback_vectors,                                          ║
║      query_text = title + " " + summary,                                     ║
║      n = 3                                                                   ║
║    )                                                                         ║
║    → for each match:                                                         ║
║        signal = "skip"        → penalty = 0.4                               ║
║        signal = "irrelevant"  → penalty = 0.2                               ║
║    → apply max penalty found                                                 ║
║                                                                              ║
║  STEP 3 — Hard exclusion check                                               ║
║    if article text overlaps news_exclusions (e.g "gossip")                  ║
║    → DROP immediately, no further processing                                 ║
║                                                                              ║
║  STEP 4 — Final score + threshold                                            ║
║    final_score = base_score - penalty                                        ║
║    final_score >= 0.3  → KEEP                                                ║
║    final_score <  0.3  → DROP                                                ║
║                                                                              ║
║  ─────────────────────────────────────────────────────────────────────────  ║
║  STATE OUT:                                                                  ║
║    filtered_articles → list[dict]                                            ║
║      each: + similarity_score                                                ║
║              feedback_penalty                                                ║
║              matching_profile_chunks  (top 3)                                ║
║              similar_feedback_articles (for reasoning awareness)             ║

║  ┌─────────────────────────────────────────────────────────────────────┐     ║
║  │  For each filtered article:                                         │     ║
║  │                                                                     │     ║
║  │  PRE-STEP — Find semantically connected past articles               │     ║
║  │    Chroma → query_similar(                                          │     ║
║  │      collection = article_vectors,                                  │     ║
║  │      query_text = title + summary,                                  │     ║
║  │      n = 5                                                          │     ║
║  │    )                                                                │     ║
║  │    → returns top 5 past article IDs (URLs)                         │     ║
║  │                                                                     │     ║
║  │    SQLite → execute_query(                                          │     ║
║  │      "SELECT title, summary, urgency, reasoning,                   │     ║
║  │       published_at FROM articles WHERE url IN (?,...)"             │     ║
║  │    )                                                                │     ║
║  │    → fetches full rows for those 5 articles                        │     ║
║  │    → these become the "connection context"                         │     ║
║  │                                                                     │     ║
║  │  SEQUENTIAL THINKING CHAIN                                         │     ║
║  │  LLM receives:                                                      │     ║
║  │    • Full profile (from state)                                      │     ║
║  │    • Current article (title + summary + source + date)             │     ║
║  │    • Top 3 matching profile chunks (from filter step)              │     ║
║  │    • Top 5 connected past articles (title + summary + urgency)     │     ║
║  │    • Feedback penalty + similar skipped articles                   │     ║
║  │                                                                     │     ║
║  │  think("What is this article actually about?")                     │     ║
║  │    ↓                                                                │     ║
║  │  think("Which dimensions of this person's life does it touch?      │     ║
║  │         Professional? Personal? Location? Tech interests?")        │     ║
║  │    ↓                                                                │     ║
║  │  think("He is an AI/ML engineer at Infosys Mysore — does          │     ║
║  │         his specific role, company, or location make               │     ║
║  │         this more or less relevant?")                              │     ║
║  │    ↓                                                                │     ║
║  │  think("Do any of the 5 past articles connect to this?             │     ║
║  │         Is the combined picture more urgent than                   │     ║
║  │         this article alone?")                                      │     ║
║  │    ↓                                                                │     ║
║  │  think("User previously skipped similar content —                  │     ║
║  │         does that lower urgency despite the match?")               │     ║
║  │    ↓                                                                │     ║
║  │  think("What is the real-world impact on this specific             │     ║
║  │         person — not people in general?")                          │     ║
║  │    ↓                                                                │     ║
║  │  think("Final verdict:                                             │     ║
║  │         relevance_score 0.0–1.0                                    │     ║
║  │         urgency 1–5                                                │     ║
║  │         reasoning (2–3 sentences, personal)                        │     ║
║  │         connected_to [{title, published_at}] or []")              │     ║
║  └─────────────────────────────────────────────────────────────────────┘     ║
║                                                                              ║
║  URGENCY SCALE                                                               ║
║  ─────────────────────────────────────────────────────────────────────────  ║
║  1 → mildly interesting, no action needed                                    ║
║  2 → worth knowing, save but don't alert                                     ║

════════════════════════════════════════════════════════════════════════════════
  FEEDBACK HANDLER  (separate always-on background process)
════════════════════════════════════════════════════════════════════════════════

  Runs continuously alongside the graph (not part of it)
  Tools: Telegram(get_callbacks) · SQLite(save_feedback) · Chroma(upsert_embedding)

  Loop every 30s:
    │
    ├── Telegram → get_callbacks()
    │     Parses lines: signal=skip  article_id=42  from=@vaslin
    │
    ├── For EACH callback:
    │     SQLite → save_feedback(article_id, signal, from_user)
    │
    └── If signal == "irrelevant" OR "skip":
          SQLite → execute_query(
            "SELECT title, summary, url FROM articles WHERE id = ?"
          )
          Chroma → upsert_embedding(
            collection = "feedback_vectors",
            id         = "{url}:{signal}",
            document   = title + " " + summary,
            metadata   = {signal, username, received_at}
          )
          → Now influences future relevance_filter_node runs
          → "skip" applies 0.4 penalty · "irrelevant" applies 0.2 penalty


════════════════════════════════════════════════════════════════════════════════
  HOW THE SYSTEM IMPROVES OVER TIME
════════════════════════════════════════════════════════════════════════════════

  Run 1    : feedback_vectors empty → no penalties
             article_vectors empty  → no connections found

  Run 5    : a few feedback signals → light penalties on similar topics
             article_vectors growing → some connections found in reasoning

  Run 30   : rich feedback history → filter drops most irrelevant articles
             article_vectors dense  → reasoning finds multi-hop connections
             alerts feel personal   → system has learned your exact taste

  Self-corrects without any manual tuning.
  Only improves. Never degrades.

  STATE SCHEMA
════════════════════════════════════════════════════════════════════════════════

  class NewsState(TypedDict):
      profile            : dict         # full flat profile from SQLite
      profile_chunks     : list[dict]   # all profile_vectors from Chroma
      search_queries     : list[str]    # LLM-generated DDG queries
      raw_articles       : list[dict]   # fetched + deduped + unseen
      filtered_articles  : list[dict]   # passed relevance filter
                                        #   + similarity_score
                                        #   + feedback_penalty
                                        #   + matching_profile_chunks
                                        #   + similar_feedback_articles
      scored_articles    : list[dict]   # after deep reasoning
                                        #   + urgency
                                        #   + relevance_score
                                        #   + reasoning
                                        #   + connected_to
      alert_articles     : list[dict]   # urgency >= 3 (subset of scored)
      errors             : Annotated[list[str], operator.add]
